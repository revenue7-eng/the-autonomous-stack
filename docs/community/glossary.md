# Glossary

## Autonomy Level (A0–A3)

A scale that measures how independent a technology is from cloud and internet.

| Level | Description |
|-------|-------------|
| **A0** | Cloud‑bound – requires constant internet and external accounts. |
| **A1** | Online‑dependent – works offline partially but needs occasional external services. |
| **A2** | Offline‑capable – works without internet, but may lack data export or recovery. |
| **A3** | Fully autonomous – offline‑first, local data, full export, pause, exit, recoverability. |

## Transparency Level (T0–T2)

A scale that measures architectural openness.

| Level | Description |
|-------|-------------|
| **T0** | Closed – no public documentation; source code not available. |
| **T1** | Documented – architecture described in whitepapers, but source may be closed. |
| **T2** | Open‑source – source code available under OSI‑approved license; fully auditable. |

## whose.world Criteria

- **Pause** – ability to stop the system at any moment.
- **Exit** – ability to leave with all your data.
- **Recoverability** – ability to roll back to a previous state.
- **Visibility** – ability to inspect how the system works.
- **External Dependencies** – reliance on third‑party cloud services.

These criteria are used in the [Philosophical Assessment](../philosophy.md) and reflected in the Autonomy and Transparency levels.
