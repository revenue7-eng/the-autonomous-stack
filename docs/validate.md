# Validate the Model

TAS ratings are only worth something if two people can independently evaluate the same technology and arrive at the same score. Right now, all assessments come from one source. That needs to change.

---

## The challenge

Pick any technology — one already in our catalog, or one that isn't — and evaluate it using [Assessment Criteria v1.0](catalog/assessment-criteria.md). Use the [assessment template](assessment-template.md) to structure your work.

If your score matches ours: the model works.
If it doesn't: tell us why. That's even more valuable.

---

## What we provide

**Formal criteria** — every test is a yes/no question about an observable fact, not an opinion. Two people following the same checklist should get the same result. Read the full [Assessment Criteria v1.0](catalog/assessment-criteria.md).

**Pilot assessments** — four technologies evaluated under v1.0, showing exactly how we applied the criteria:

| Technology | Score | Type | Difficulty |
|------------|-------|------|------------|
| [Syncthing](catalog/syncthing.md) | S3/3 — D5/5 — A3/T2 | File sync (P2P) | Clean case |
| [KeePass](catalog/keepass.md) | S3/3 — D5/5 — A3/T2 | Password manager (local) | Clean case |
| [Nextcloud](catalog/nextcloud.md) | S3/3 — D4/5 — A3/T2 | Cloud platform (self-hosted) | Moderate |
| [Signal](catalog/signal.md) | S3/3 — D4/5 — A3/T2 | Messaging (centralized E2E) | Borderline |

**Template** — a blank [assessment template](assessment-template.md) you can copy and fill in.

---

## How to participate

1. Read [Assessment Criteria v1.0](catalog/assessment-criteria.md)
2. Pick a technology
3. Fill in the [assessment template](assessment-template.md)
4. Submit as a [GitHub Issue](https://github.com/revenue7-eng/the-autonomous-stack/issues/new?labels=independent-assessment&title=Independent%20assessment%3A%20%5Btechnology%20name%5D) or Pull Request

We will publish every independent assessment — whether it agrees with ours or not.

---

## What counts as success

- **A-level and T-level match** → criteria are clear enough to be reproducible
- **D-score within ±1** → acceptable, diagnostic questions involve more judgment
- **A or T differ** → criteria need refinement, and your assessment helps us find where

Every disagreement makes the model better.

---

## Recognition

Independent assessments are credited in the catalog:

> _Assessment criteria: v1.0. Last verified: 2025-05._
> _Independent validation by @username (date)._

The first external assessments carry particular weight — they establish that TAS is not one person's opinion, but a reproducible framework.
